# 复古相机 App — 规格说明（SDD）

## 1. 背景与目标

在 Android 上提供一款相机应用：可选择复古相机风格、多种滤镜，并支持带倒计时的录像开始。

**开发与调试**：使用 [Expo](https://docs.expo.dev/) 52 版本与 **Expo Go** 在真机上快速迭代；日常联调以 **局域网（LAN）** 为主，保证本机与手机在同一 Wi‑Fi，扫码或输入 Metro 地址即可热更新预览。

## 2. 技术栈与运行方式（Expo Go + 局域网）

| 项 | 说明 |
|----|------|
| 运行时 | **Expo Go**（开发阶段）；发布形态可为 EAS Build / 独立 APK，与 V1 范围一致时仍以 Android 为主。 |
| 启动 | `npx expo start`（或 `npm start`），在终端选择 **LAN** 连接模式（非 Tunnel），确保电脑防火墙允许 Node/Metro 端口（默认 **8081**）。 |
| 手机 | 安装 Expo Go，与开发机 **同一局域网**；用 Expo Go 扫描终端二维码，或手动输入 `exp://<电脑局域网IP>:8081`。 |
| 验收（联调） | 修改 JS/资源后，手机端 **Fast Refresh** 生效；相机/相册等权限在 Expo Go 内按系统弹窗授权即可验证流程。 |

**注意**：若后续接入 Expo Go 不支持的自定义原生模块，需改为 [Development Build](https://docs.expo.dev/develop/development-builds/introduction/)，本规格 V1 以 **Expo Go 可运行的 API**（如 `expo-camera`、`expo-media-library` 等）为约束。

## 3. 范围（V1）

| 能力 | 说明 |
|------|------|
| 拍照 | 预览、快门、保存到系统相册（需授权） |
| 相机模式 | **6** 种底层色调（Skia **ColorMatrix** 完整 4×5），切换后预览与成片风格明显变化；默认 **自动胶片** |
| 滤镜 | **8** 款上层色彩矩阵，叠加在相机模式之上；成片为 **滤镜矩阵 × 相机矩阵**（实现顺序以代码 `combinedMatrix` 为准） |
| 延时录像 | 开始录像前可选 **0 / 3 / 5 / 10 秒** 倒计时，再调用系统录像 |

## 4. 非目标（V1 不做）

- 实时给**已导出视频文件**做胶片级调色（工程量大，需原生管线或 FFmpeg；V1 录像为设备默认编码结果）。
- iOS 商店发布流程（代码可扩展，但本仓库以 Android APK 为主）。

## 5. 用户故事与验收标准

1. **权限**  
   - 首次进入若未授权相机/麦克风（录像）/相册写入，应出现系统授权或清晰提示；拒绝后仍可进入界面但功能不可用并提示原因。  
   - *验收*：拒绝相机后无法预览；授权后可预览。

2. **相机模式**  
   - 共 **6** 种预设（见 §3.1），每种绑定完整 Skia 色彩矩阵与预览色调示意。  
   - 切换预设时预览区有可见变化；拍照后成片与当前「**相机 + 滤镜**」一致。  
   - *验收*：同一场景切换相机模式，保存的两张照片观感明显不同。

3. **滤镜**  
   - **8** 款可切换滤镜（见 §3.2），全部为非单位矩阵；与相机矩阵组合后成片色彩可辨认，任意两款滤镜在同一相机下差异可感知。  
   - *验收*：固定相机模式下切换滤镜，成片变化可辨认。

4. **延时录像**  
   - 可选倒计时秒数；点击开始后先显示 **3、2、1**（或对应秒数）再开始录制；再次点击停止并保存。  
   - *验收*：选 5 秒时，录制开始明显晚于按钮按下约 5 秒。

5. **Expo Go + 局域网调试**  
   - *验收*：开发机与手机同网，使用 LAN 模式启动后，Expo Go 能打开项目并完成拍照/录像/保存主流程；无必须依赖自定义原生插件才能跑通的路径。
